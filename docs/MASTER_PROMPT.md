## THE BRIDGE PROTOCOL — HOW AGENTS HAND OFF

When Claude completes a task:
1. Write changed files
2. Run: `git add [changed files] && git commit -m "[task description]" && git push origin main`
3. Write one-line completion note to `docs/KIRO_LOG.md`
4. If a follow-up task is needed for Kiro, write it to `docs/KIRO_LOG.md` as `NEXT: [task]`

When Kiro picks up:
1. Read `docs/KIRO_LOG.md`
2. Execute the `NEXT:` task in autopilot
3. Overwrite `NEXT:` with `DONE: [timestamp]` when complete

When Maurice checks in:
1. He reads `docs/KIRO_LOG.md` — one file, full status
2. He does not ask what happened — the log tells him
3. He does not approve individual steps — only CEO decisions
