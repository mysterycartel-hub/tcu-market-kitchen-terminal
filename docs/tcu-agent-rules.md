# TCU Agent Rules

## Product Rules
- This is not a broker.
- This is not a signal app.
- This is not financial advice.
- This is practice and education only.
- Never block the student.
- Chef warns, student decides.
- Every mistake becomes a lesson.
- Practice / Simulation mode must remain visible.
- BUY and SELL buttons must remain visible and open Kitchen Tickets.
- Do not create live broker claims.
- Do not create fake live data claims.
- Do not create fake calendar connection claims.

## UI Rules
- Keep the interface educational and coach-driven.
- Use clear practice disclaimers.
- Preserve all existing terminal logic before making UI upgrades.
- Avoid language like guaranteed, signal, trade executed, broker connected, live data connected, or cannot continue.

## Experience Rules
- Reveal risk clearly.
- Warn clearly on missing setup elements.
- Save learning entries for both normal and Practice Anyway tickets.
- Let the student decide whether to continue.
- Use roadmap language consistently:
  - Bias = Kitchen Direction
  - Liquidity = Flow
  - AOI = Prep Zone
  - Delivery = How Price Moves
  - Confirmation = Candle Proof
  - Entry = The Pass
  - Targets = Tables Served
  - Management = Protect the Plate

## Documentation Protocol
Before editing code, read the build bible and all canonical docs. Audit assets and folders first. Do not invent brand rules or remove required behavior.
