# AI Content Ecosystem Architecture

## Layer Purpose
This architecture defines how MysterMyself uses AI to convert market education work into repeatable content, publishing assets, audience growth, and product outcomes.

This is not isolated from the platform.
The TCU Terminal is one stage inside the larger MysterMyself / Scott-King Coast / Market Marina ecosystem.

## Parent Context
- Top identity: Maurice Scott / MysterMyself
- Ecosystem map: Scott-King Coast
- Learning district: Market Marina
- Source inventory district: Library Vault

## End-to-End Ecosystem Flow
Research signal -> TCU chart lesson -> Kitchen Ticket -> Journal -> Mistake Review -> Video Lesson -> Publishing Package -> Subscriber Funnel -> Product System

## System Components
1. Research Signal Intake
- Inputs: chart observations, market themes, concept triggers, mission context.
- Output: normalized lesson candidate with topic, symbol, bias, and training objective.

2. TCU Lesson Session
- Inputs: selected symbol, chart setup, execution plan, coach notes.
- Output: practice-first lesson state and ticket-ready session record.

3. Knowledge Capture
- Inputs: Kitchen Ticket, Journal, Mistake Review.
- Output: structured learning dataset for coaching and content generation.

4. Video Lesson Generation
- Inputs: approved lesson state + reflection notes.
- Output: scene-ready storyboard payload for the video factory.

5. Publishing Package Generation
- Inputs: video script, headline angle, audience segment.
- Output: title options, hook lines, thumbnail direction, short/long captions, CTA stack.

6. Subscriber Funnel Sync
- Inputs: publishing links and CTA logic.
- Output: Beehiiv subscriber capture, newsletter sequence entry, audience tagging.

7. Product System Routing
- Inputs: subscriber behavior and lesson category.
- Output: routing into products such as TCU paths, Newsletter Ready Desk, and Opportunity List.

## AI Operating Rules
- Classify request intent first: Research, Audit, Compare, Store, Monitor, or Build.
- Enter Build mode only when explicitly requested by Maurice.
- Use existing canon/assets before asking for new uploads.
- Keep all outputs educational and practice-first.
- No broker execution, no live trading claims, no profit promises.

## TCU Terminal Connection
The terminal acts as the lesson capture engine for the ecosystem.
It supplies the structured learning state used by:
- Video Factory payload creation
- Publishing package generation
- Subscriber funnel messaging
- Product system categorization

## Library Vault Connection
All generated and imported artifacts should route into Library Vault categories:
Assets, Characters, Brands, Products, Websites, Apps, Repositories, Content, SOPs, Research, Archive.

## Build Direction
When extending this layer, prioritize:
1. Schema consistency across lesson, ticket, and journal records.
2. Deterministic content payloads for video and publishing outputs.
3. Traceable links from each content asset back to the original lesson state.
